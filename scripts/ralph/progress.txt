## Codebase Patterns
- Prisma client singleton lives in `src/config/db.ts` (reuse via `import prisma from "../config/db"`) — do NOT instantiate a second `PrismaClient`.
- Tests use `vitest` (config at `vitest.config.ts`). Test files live in `tests/` at repo root, not colocated.
- Pre-existing typecheck error in `src/scripts/seed-knowledge.ts` (missing `langchain/text_splitter` import path). Treat as known noise — only gate on typecheck delta for your changes, or fix it as part of US-003.
- `KnowledgeDocument.embedding` is `Unsupported("vector(1536)")` in Prisma — writes/reads must use `$executeRawUnsafe` / `$queryRawUnsafe` with `::vector` casts. Never interpolate user input.
- `langchain@^1.3.3` is the v1 API — use LCEL `RunnableSequence` / `ChatPromptTemplate`, NOT the deprecated `ConversationalRetrievalChain`.
- Project uses `dotenv` for secrets; fail-fast on missing keys at boot rather than returning `undefined` deep in a call chain.
- `.env` `DATABASE_URL` points to Supabase (production) — NEVER run `prisma migrate dev` with default env. Use local Docker DB (`alphatoca_db` on :5433, creds `admin`/`admin_pwd`) via inline env overrides.
- `prisma migrate dev` refuses to run non-interactively in this harness. Use this two-step workaround: (1) `prisma migrate diff --from-migrations ./prisma/migrations --to-schema-datamodel ./prisma/schema.prisma --shadow-database-url <local_shadow> --script` to emit SQL; (2) write it to `prisma/migrations/<TS>_<name>/migration.sql`; (3) `prisma migrate deploy` with local env overrides to apply.
- Shadow DB (`alphatoca_shadow`) exists on the local Docker postgres for migration diffs; pgvector extension already enabled on both local DBs.
- `KnowledgeDocument` now has ingestion-tracking fields: `sourcePath`, `chunkIndex`, `contentHash`, `createdAt`, `updatedAt`. Compound unique index on `(sourcePath, chunkIndex)` — use this for upserts. Index on `contentHash` for hash-skip lookups.
- The existing `prisma/seed.ts` + `prisma/generators.ts` insert `KnowledgeDocument` rows via `$executeRawUnsafe` without the new NOT NULL columns — they will fail at runtime post-migration. US-003 should replace the seed path or update the generator to supply `source_path`, `chunk_index`, `content_hash`, `updated_at`.
