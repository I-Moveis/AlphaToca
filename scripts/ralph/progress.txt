## Codebase Patterns
- `src/app.ts` mounts `express.static` BEFORE the `authStack` so static assets bypass `checkJwt`. Any new public file route must follow the same ordering.
- Static-file responses are customized via `express.static(root, { setHeaders })` — the callback signature is `(res: ServerResponse) => void`, NOT `(res: Response) => void`. Import `ServerResponse` from `node:http` (`import type { ServerResponse } from 'http'`) for the type.
- Image upload URLs are stored relative as `/uploads/<propertyId>/<file>.ext` (see `propertyImageStorageService.savePropertyImages`). Contract PDFs follow the same shape under `/uploads/contract-<id>/<file>.pdf` (see `contractDocumentStorageService.saveContractDocument`). DO NOT migrate these to absolute URLs — the backend dual-mount in `/uploads` and `/api/uploads` is the supported way to make them resolve.
- Tests in `tests/` follow a `supertest` + `request(app)` pattern. `tests/contractPdf.test.ts` is the canonical reference for tests that need a real fixture file under `uploads/` — it uses `beforeAll`/`afterAll` to mkdir/rm a temp dir under `UPLOADS_ROOT = path.resolve(__dirname, '../uploads')`.
- Tests that import `src/app` must set Firebase env vars (FIREBASE_PROJECT_ID, FIREBASE_CLIENT_EMAIL, FIREBASE_PRIVATE_KEY) BEFORE the `import app from '../src/app'` line, because `validateAuthConfig()` runs at module load.
- `npm run build` runs `tsc` (no emit issues at HEAD). Use it as the typecheck gate before commit.
- For tests that hit ONLY the static mounts (anything under `/uploads` or `/api/uploads`), no auth-middleware mocking is needed — those mounts sit ahead of `authStack` in `src/app.ts`. Just set the Firebase env vars and import `app`; the firebase init in `src/config/firebase.ts` swallows credential errors in a try/catch and the unrelated checkJwt is never invoked. Compare with `contractPdf.test.ts` which DOES mock `authMiddleware` because it hits a protected endpoint.
- Before editing `scripts/ralph/prd.json`, validate the file with `jq . scripts/ralph/prd.json` — prior Ralph cycles have shipped malformed JSON (a missing `},{` between user stories), and editing on top of broken JSON quietly produces non-parseable output.
