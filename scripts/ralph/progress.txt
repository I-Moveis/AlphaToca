- Idempotency idiom for any `scripts/db-migration/` shell script that edits a config file: preserve the pristine original via `cp -p file file.bak` only if `file.bak` does not exist, then `sed -i` without a `.bak` suffix (or with a deleted `.tmp` scratch). Re-runs must never overwrite the pristine backup.
- Postgres DDL idempotency: `CREATE DATABASE` does NOT support `IF NOT EXISTS`; gate with `SELECT 1 FROM pg_database WHERE datname='…'` first. `CREATE ROLE` should be wrapped in a `DO $$ … IF NOT EXISTS (SELECT 1 FROM pg_roles WHERE rolname='…') THEN CREATE ROLE … END IF; END $$;` block.
- Apt PG paths on Debian/Ubuntu after `postgresql-16` install: configs at `/etc/postgresql/16/main/{postgresql,pg_hba}.conf`, service unit `postgresql`. Don't try to discover paths dynamically; this is the convention every operator script assumes.
- Modern apt repo signing: keyrings live at `/etc/apt/keyrings/`, source entries use `[signed-by=…]`. Do not use `apt-key add` (deprecated). The 01-provision-postgres.sh PGDG setup follows this pattern — copy it for any future apt-repo additions.
- `pg_dump --jobs=N` is incompatible with `--format=custom`; parallel dump only works with `--format=directory`. The PRD AC for US-003 listed both — scripts here resolve the conflict by keeping `--format=custom` (single-file, easier scp/checksum) and dropping `--jobs`. Parallel restore (`pg_restore --jobs=4`) IS supported with custom-format dumps and is used in 03-restore-local.sh. Document the choice in a header comment so future readers don't "fix" it back.
- Timestamps in operator artefacts (dump files, backups) use UTC ISO basic format: `date -u +%Y%m%dT%H%M%SZ` (e.g., `20260509T034512Z`). Avoid local time / colons — both confuse downstream tools and SCP filenames.
- Prisma model → physical table mapping: every Prisma model in `prisma/schema.prisma` carries a `@@map("snake_case")` directive — the SQL table name is the `@@map` value, NOT the Prisma model name. Critical mappings: `User`→`users`, `Property`→`properties`, `PropertyImage`→`property_images`, `Conversation`→`conversations`, `ConversationMessage`→`conversation_messages`, `RentalProcess`→`rental_processes`. There is NO `Lead` / `leads` table in the current schema (the PRD AC for US-004 names it but it does not exist) — defensive scripts must use `to_regclass('public.<table>')` to handle the absence gracefully.
- Bash idiom for "capture rc without aborting under `set -e`": wrap the call in `set +e ... set -e` rather than `|| true` (which discards the rc). Used for `pg_restore` (warnings cause non-zero exit which is non-fatal) and per-extension `CREATE EXTENSION` (Supabase-only extensions fail on a vanilla postgresql-contrib-16 install). Pattern:
    ```
    set +e
    pg_restore … ; rc=$?
    set -e
    ```
- Supabase-only Postgres extensions to expect in any production dump (NOT shipped with `postgresql-contrib-16`, so `CREATE EXTENSION` will fail and the application schema does NOT depend on them): `pg_graphql`, `pgjwt`, `vault`/`supabase_vault`, `pgsodium`, `pg_net`, `pgaudit`, `wrappers`. Any future script that touches extensions should warn-and-continue on these rather than abort.
- `pg_restore --list` extension lines are formatted `<id>; <oid> <oid> EXTENSION - <name>` — the extension name is the LAST whitespace-delimited field. There is also an `EXTENSION COMMENT` line for each that must be filtered out (`grep -v 'EXTENSION COMMENT'`) to avoid double-creating.
- For dynamic per-table `count(*)` from a label list, the cleanest single-statement pattern is a PL/pgSQL `DO` block iterating a `(VALUES …)` rowset, with `to_regclass('public.' || quote_ident(tbl))` to skip missing tables and `EXECUTE format('SELECT count(*) FROM public.%I', tbl) INTO cnt`. Output via `RAISE NOTICE` lands on stderr (psql shows it) and stays as a single SQL command per the AC.
- URL-encoding a password for a Postgres connection string from bash uses python3's stdlib (no node, no jq dependency): `python3 -c 'import urllib.parse,sys;print(urllib.parse.quote(sys.argv[1],safe=""))' "$PASSWORD"`. Pass `safe=""` so `/` and `@` are escaped — they're legal in passwords but break URL parsing. Encoded output never contains `|`, so `sed` can use `|` as the delimiter and skip escaping `/` in the URL itself.
- For `sed -i` on a connection-string substitution, the URL contains `/` so the conventional `s/…/…/` delimiter forces escaping. Use a `'|'` delimiter (`s|…|…|`) — the URL-encoded password from `urllib.parse.quote(safe="")` cannot produce `|`, so the delimiter is collision-safe. Pattern used in 04-cutover-env.sh.
- `.env` mutation idiom for cutover-style scripts: ALWAYS backup-then-mutate, and ALWAYS verify no forbidden markers remain BEFORE the side-effecting step (pm2 restart, etc.). The 04-cutover-env.sh sequence is: cp → sed → grep-verify-fail-fast → pm2 restart. Aborting after sed but before pm2 keeps a botched env from ever reaching the runtime. Backup name convention: `<file>.pre-supabase-cutover.bak`, refused if it already exists (no silent overwrite).
- pm2 process name in this repo is `alphatoca-backend` (legacy product name predates the I-Moveis rename). Confirmed via `plan_hospedagem.md:65` (`pm2 start dist/app.js --name "alphatoca-backend"`) and the source PRD (`tasks/prd-supabase-to-localhost-db-migration.md`). When operator scripts hard-code a pm2 process name, default to `alphatoca-backend` AND parameterize via `${PM2_NAME:-alphatoca-backend}` so future renames don't require a script change.
- Operator-script tty detection idiom: `if [[ ! -t 0 ]]; then fail "stdin is not a tty"; fi` before `read -s`. Without this, a CI/automation re-run with no input silently blocks. The pattern is used in 04-cutover-env.sh's password prompt when `$2` is omitted.
