- **Learnings for future iterations:**
  - Docker Compose v2 `profiles: ["nome"]` é a forma idiomática de tornar serviços opt-in. `docker compose up -d` ignora serviços com profile; só sobem quando você passa `--profile nome` ou `COMPOSE_PROFILES=nome`.
  - Volumes nomeados (`postgres_data`) exigem declaração no bloco top-level `volumes:` mesmo quando referenciados em apenas 1 serviço — caso contrário Compose trata como bind mount com path relativo.
  - Quando o serviço Docker Postgres (porta host 5433) coexiste com Postgres nativo (porta host 5432), as duas instâncias são independentes — DATABASE_URL em .env continua apontando para 5432 (nativo); para usar o Docker, dev precisa editar manualmente DATABASE_URL para 5433. Esse trade-off está documentado no comentário do compose e (futuramente) será explicitado no README (US-004).
